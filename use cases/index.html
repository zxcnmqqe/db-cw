# Сценарії використання

# UserSignUp

ID UserSignUp
Назва: Реєстрація нового облікового запису
Учасники: Користувач, система
Передумови: Користувач не має облікового запису
Результат: Новий обліковий запис
Виключні ситуації: - SignUpException_InvalidData - введені користувачем дані не коректні
- SignUpException_AccAlreadyExist - обліковий запис по вказаному Email вже існує
- SignUpException_InvalidEmail - введений Email не існує

# UserSignIn

ID UserSignIn
Назва: Автентифікація у існуючий обліковий запис
Учасники: Користувач, система
Передумови: - Користувач має обліковий запис
- Користувач не авнтентифікований
Результат: Користувач автентифікований у системі
Виключні ситуації: - SignInException_InvalidData - введені користувачем дані не коректні
- SignInException_AccDoesn`tExist - обліковий запис по вказаному Email не існує
- SignInException_InvalidPassword - введений користувачем пароль не правильний
Основний сценарій:
  1. Користувач заповнює поля у формі для входу
  2. Користувач натискає кнопку "Увійти"
  3. Система перевіряє введені дані(SignInException_InvalidData, SignInException_AccDoesn`tExist, SignInException_InvalidPassword)
  4. Система автентифікує користувача

# UserUpdate

ID UserUpdate
Назва: Редагування існуючого облікового запису
Учасники: Користувач, система
Передумови: Користувач автентифікований у системі
Результат: Дані облікового запису менеджера/користувача змінені
Виключні ситуації: Введені Користувачем дані не коректні UserUpdateException_InvalidData
менеджер/користувач не авторизований UserUpdateException_Unauthorized

# UserAdd

ID UserAdd
Назва: Додавання користувача до проєкту
Учасники: Адміністратор, користувач, система
Передумови: - Адміністратор автентифікований у системі
- Користувач якого додають до проєкту існує
Результат: До проєкту додано користувача
Виключні ситуації: - UserAddException_InvalidData - введені адміністратором дані не коректні
- UserAddException_Unauthorized - адміністратор не має прав на додавання користувача

# UserDelete

ID UserDelete
Назва: Видалення користувача з проєкту
Учасники: Адміністратор, користувач, система
Передумови: - Адміністратор автентифікований у системі
- Користувач якого видаляють є в проєкті
Результат: З проєкту видалено користувача
Виключні ситуації: - UserDeleteException_InvalidData - введені адміністратором дані не коректні
- UserDeleteException_Unauthorized - адміністратор не має прав на видалення користувача
Основний сценарій:
  1. Адміністратор обирає проєкт у розділі "Проєкти"
  2. Адміністратор обирає користувача у розділі "Користувачі"
  3. Адміністратор натискає кнопку "Видалити користувача"
  4. Система запитує чи менеджер дійсно хоче видалити користувача
  5. Адміністратор натискає кнопку "Видалити"
  6. Система перевіряє введені дані(UserDeleteException_InvalidData, UserDeleteException_Unauthorized)
  7. Система видаляє користувача з проєкту

- TaskAddException_Unauthorized - менеджер не має прав на додавання задачі

# TaskUpdate

ID TaskUpdate
Назва: Редагування задачі в проєкті
Учасники: Менеджер, система
Передумови: - Менеджер автентифікований у системі
- Проєкт в якому редагують задачу існує
Результат: Задачу відредаговано
Виключні ситуації: - TaskUpdateException_InvalidData - введені менеджером дані не коректні
- TaskUpdateException_Unauthorized - менеджер не має прав на змінення задачі
Основний сценарій:
  1. Менеджер обирає проєкт у розділі "Проєкти"
  2. Менеджер обирає задачу у розділі "Задачі"
  3. Менеджер натискає кнопку "Редагувати"
  4. Система відкриває вікно з формою для редагування задачі
  5. Менеджер заповнює форму та натискає кнопку "Зберегти"
  6. Система перевіряє введені дані(TaskUpdateException_InvalidData, TaskUpdateException_Unauthorized)
  7. Система змінює задачу

  5. Менеджер натискає кнопку "Видалити"
